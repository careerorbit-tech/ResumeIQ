## Code of Conduct
This project and everyone participating in it is governed by the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/0/code_of_conduct/). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs
Bugs are tracked as GitHub issues. When creating a bug report, please include:
- A clear and descriptive title.
- Steps to reproduce the bug.
- Actual vs. Expected behavior.
- Environment details (Browser, OS, Node version).

### Suggesting Enhancements
We love new ideas! If you have a suggestion:
- Check existing issues to see if the feature has been suggested.
- Open a new issue with the "Enhancement" label.
- Describe the motivation and how users will benefit.

### Pull Requests
1.  **Fork** the repository.
2.  **Clone** your fork (`git clone ...`).
3.  **Create** a branch for your fix/feature.
4.  **Lint/Test** your changes.
5.  **Submit** a pull request with a clear description of the change.

## Style Guides

### Git Commit Messages
- Use the imperative mood ("Add feature" not "Added feature").
- Limit the first line to 72 characters or less.
- Reference issues and pull requests liberally.

### TypeScript / React Style Guide
- Use functional components and hooks.
- Use explicit types; avoid `any` wherever possible.
- Wrap complex logic in custom hooks.
- Follow the component-based architecture in `client/src/components`.

### Documentation Style
- Use Markdown for all documentation.
- Keep the `README.md` updated with any major system changes.
- Write clear, concise, and professional comments for complex algorithms.
